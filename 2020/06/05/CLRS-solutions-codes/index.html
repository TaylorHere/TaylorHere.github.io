2.1.-3

Consider the *searching problem*:

Input: A sequence of n numbers A=⟨a1,a2,…,an⟩A=⟨a1,a2,…,a⟩ and a value v.

Output: An index i such that v=A[i]v=A[i] or the special value NILNIL if v does not appear in A.

Write pseudocode for linear search*, which scans through the sequence, looking for vv*. Using a loop invariant, prove that your algorithm is correct. Make sure that your loop invariant fulfills the three necessary properties.

Loop invariant: At the start of each iteration of the for loop, the subarray A[1..i] consists of elements that are different than v.

Initialization: before the for loop, key holds noting, and array is not empty, subarray is empty so the proof is trivial.

Maintenance: During each loop, we we compare v with key, if they are the same, we return i, which is the correct result, otherwise , we continue to the next iteration, at the end of each loop, we know the subarray A[1..i] does not contain v, so the loop invariant holds true, incrementing i for the next iteration of the for loop preserves the loop invariant.

Termination: the loop terminated when elements has compared with v, and we found a element equal to v, or every elements has compared with v, and i equals to len(A) -1, in the other way, the elements in subarray are different than v, thus we return ’nil’, then we conclude that the entire array does not have any element equal to v. Hence the algorithm is correct.

2.2-2

Consider sorting n numbers stored in array A by first finding the smallest element of A and exchanging it with the element in A[1]. Then find the second smallest element of A, and exchange it with A[2]. Continue in this manner for the first n−1 elements of A. Write pseudocode for this algorithm, which is known as *selection sort*. What loop invariant does this algorithm maintain? Why does it need to run for only the first n−1 elements, rather than for all n elements? Give the best-case and worst-case running times of selection sort in Θ-notation.

Loop Invariant : At the every turns of the first for loop, the subarray A[1…n-1] contains of the smallest i-1 elements in array A with sorted order

Why n-1?: After n-1 iterrations ,A[n] is already the biggest element.

Running consume:

line by line cost and times:

num cost time
3 $c_1$ $n$
4 $c_2$ $n-1$
5 $c_3$ $\sum_{i=0}^{n-1}t_j$
6 $c_4$ $\sum_{i=0}^{n-1}(t_j-1)$
7 $c_5$ $\sum_{i=0}^{n-1}(t_j-1)$
8 $c_6$ $n-1$

$$
T(n) = c_1 * n + c_2 * (n-1) + c_3 \sum_{i=0}^{n-1}t_j + c_4 \sum_{i=0}^{n-1}(t_j-1) + c_5 \sum_{i=0}^{n-1}(t_j-1) + c_6 (n-1)
$$

Best case:

A is already sorted in nondecreasing order

Worst case:

A is sorted in nonincreasing order

in both cases
$$
t_j = n-i \\
\sum_{i=0}^{n-1}n-i = n^2-n-\frac{n\left(n-1\right)}{2} \\
\sum_{i=0}^{n-1}(n-i-1) = n^2-\frac{n\left(n-1\right)}{2} \\
$$
So
$$
T(n) = c_1 * n + c_2 * (n-1) + c_3 (n^2-n-\frac{n\left(n-1\right)}{2} )+ c_4 (n^2-\frac{n\left(n-1\right)}{2}) + c_5 (n^2-\frac{n\left(n-1\right)}{2}) + c_6 (n-1)
$$
Remove all Constant
$$
T(n) = n + (n-1) + n^2-n-\frac{n\left(n-1\right)}{2} +n^2-\frac{n\left(n-1\right)}{2} + n^2-\frac{n\left(n-1\right)}{2} + (n-1)
$$
Thus time consume is
$$
\Theta(n^2)
$$

2.3-3

Use mathematical induction to show that when nn is an exact power of 2, the solution of the recurrence
$$
T(n) =
\begin{cases}
2, & \text{if n = 2}, \
2T(n/2)+n, & \text{if n = 2^k, for k> 1}
\end{cases}
$$

Is $T(n)=nlog_2n$

For $n=2^1, T(n)=2=2log_22$

Suppose $n = 2^k, T(n)=nlog_2n=2^klog_2{2^k}=2^kk$

For $n=2^{k+1}$,
$$
\begin{equation} \label{eq1}
\begin{split}
T(n) & = 2T(2^{k+1}/2)+2^{k+1} \
& = 2T(2^k)+2^{k+1} \
& = 2*2^kk + 2^{k+1} \
& = 2^{k+1}(k+1) \
& = 2^{k+1}log_2{2^{k+1}} \
& = nlog_2n
\end{split}
\end{equation}
$$
By the Principle of M.I, we can say,

$T(n) = nlog_2n$, when n is an exact power of 2.
